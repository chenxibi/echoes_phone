# 2026-05-02

## 功能实现

### Smart Time Display + 真实时间感知 Toggle
- `formatSmartTime()` in `helpers.js`: 今天→HH:mm, 昨天→昨天 HH:mm, 前天→前天 HH:mm, 今年→M月D日 HH:mm, 去年及以前→YYYY年M月D日 HH:mm
- `toLocaleTimeString("en-US",...)` → `toLocaleTimeString([],...)` 让时间格式跟随设备系统 locale，两版统一
- "真实时间感知" toggle in Settings.jsx，默认ON；关闭后消息完全不显示时间戳，不显示间隙标记，不触发 Time Gap Notice
- 重新开启时 skipNextGapNotice，第一条消息恢复正常但跳过 Time Gap Notice
- Time Gap Marker: 消息间隔>1h时显示时间分隔线
- 提示文案: CN "【线下】、【小说】、【剧本】模式下推荐关闭" / EN "Recommended off for [Reality] / [Novel] / [Script] mode"
- EN 同步完成

### 转账 Bug 修复
- prompt 里 `transfer_action` 值从 `"accept"/"reject"` → `"accepted"/"rejected"`，与 TransferBubble 判断一致
- AI 之前按旧 prompt 返回 `"accept"`，前端只判断 `"accepted"`，不匹配就 fallback 到"已退还"

### 角色语音功能
- AI 可在 JSON messages 数组中用 `{"text": "...", "isVoice": true}` 发语音
- 也兼容文本标记 `[语音]xxx` / `[Voice]xxx`
- VoiceMessageBubble 已支持 isMe prop，角色语音直接渲染

### Auto-update Alert 时序 Bug 修复
- `runGenerator` 不再自己弹通用 toast，改为返回 true/false
- triggerAppEvents 里日记/账单/浏览器/位置更新四个分支改为检查返回值后才弹 toast
- 位置更新加了 `smartWatchLocations.length > 0` 初始化检查

### 论坛自动发帖修复
- useEffect 依赖太多导致 `window.__forumGenerateChatEventPost` 频繁重建 → 改用 useRef 存函数引用
